# The suite is a static judgment layer: live-execution stays out, temporal memory is in-scope-by-philosophy but deferred behind a defined seam

The *Strategic Synthesis and Opportunities for Remediation* section of a market analysis of judgment-layer tools names four gaps in this suite. Two of them — the **Governance Gap** and the **Autonomy Gap** — are already answered: governance by [ADR-0007](0007-sentinel-sacred-path-fail-override.md)'s Sacred-Path FAIL Override (J-Rig-style binary rigor on designated paths), and autonomy by `debug-test` routing locator/timing failures to the Playwright healer and logic failures to `diagnosing-bugs`. The remaining two — the **Execution Gap** and the **Temporal Gap** — have no decision recorded, so they keep reading as unclaimed deficiencies every time someone re-runs the analysis. This ADR rules on both, so they stop being re-litigated.

## Execution Gap → out of scope, delegated across the existing routing seam

**The proposal:** augment `debug-test` to drive live browsers (Browserbase / Stagehand), spin up headless sessions, navigate to the failure state, and extract the live DOM — so the suite can catch UI regressions static analysis can't see.

**Decision: out of scope.** The suite is deliberately a static-analysis + judgment layer; "judgment, not execution" is its whole thesis (see `ARCHITECTURE.md` / `CONTEXT.md`). Live browser execution is the Execution Layer's job, and mature tools already own it — Playwright's own agents, Browserbase. `debug-test` already reaches execution *across a seam*: it routes locator/timing failures to the Playwright healer (which self-heals) and value/logic failures to `diagnosing-bugs`. Absorbing a browser-driving stack would (a) force the suite to run infrastructure it deliberately doesn't, (b) duplicate those mature tools, and (c) collapse the judgment/execution seam that is the suite's competitive moat. So execution is *reached by routing, not owned.*

This is the same reasoning that answers the report's **Autonomy Gap**: self-healing is delegated to the Playwright healer, not natively generated by the suite. The suite stops at diagnosis + routing on purpose. Native generative test-rewriting is likewise out of scope, for the same seam-preserving reason.

## Temporal Gap → in-scope-by-philosophy, deferred behind a defined seam

**The proposal:** a `projectmem`-style append-only event log so `threat-model` and `sentinel` reference repository-specific history (past bugs, failed patterns, production rollbacks) instead of pure heuristics.

**Decision: in scope by philosophy, deferred in practice, with the seam shape fixed now.** Unlike execution, temporal memory *is* judgment, not execution, and a deterministic, plain-text, append-only log aligns with the suite's anti-hallucination stance (the same instinct behind `audit-test`'s proof-by-execution and `prune-tests`' boundary checks). It is a legitimate future capability — but it deserves its own design pass, not a rushed bolt-on, so it is deferred here with a defined shape rather than left open:

- **Substrate:** a local, repo-relative, plain-text, append-only findings log — not session memory. It records the verdicts of `audit-test`, `bug-report`, and `debug-test` (what was proven hollow, what broke, how it was reproduced), keyed by code/test path.
- **Readers:** `threat-model` and `sentinel` consume it as an *additional input*. A path with a previously-logged production rollback or repeated false-confidence history raises its risk profile; it could also inform which paths a reviewer marks `--sacred`.
- **Keep it separate from harness memory.** This suite-level log is about test/code findings in *this* repo, not about a Claude session. The two must not be conflated.

**Guardrails, so temporal memory doesn't drift into the failure modes the other ADRs guard against:**
- It stays **evidence for judgment**, never a second scoring system — no "history score." (ADR-0002: Sentinel stays categorical; history informs the verdict, it doesn't compute one.)
- It never **silently gates**. Any hard consequence flows through the existing user-designated mechanism (ADR-0007: the user says what's sacred; memory doesn't unilaterally promote a path).

## Consequences

- Each of the report's four Strategic-Synthesis items now has a recorded decision: **#1 Execution — out (delegated via routing)**, **#2 Temporal — deferred behind the seam above**, **#3 Governance — closed ([ADR-0007](0007-sentinel-sacred-path-fail-override.md))**, **#4 Autonomy — delegated to the Playwright healer (this ADR)**. Future reviews cite this ADR instead of re-flagging the same two gaps.
- No browser-driving or cloud-session dependency enters the suite; `debug-test` keeps reaching execution only through the healer / `diagnosing-bugs` seam.
- If temporal memory is picked up, it starts from the substrate/readers/guardrails shape above, not a blank page — and gets its own ADR recording the concrete schema and write triggers when it ships.
